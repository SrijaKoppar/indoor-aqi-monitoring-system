================================================================================
API ENDPOINTS
================================================================================

Frontend → Backend Communication:

GET /health
  Purpose: Check if backend is available
  Response: {"status": "healthy", "models_loaded": true}

POST /api/predict
  Purpose: Get CO prediction from sensor readings
  Request: {co_gt, nox_gt, no2_gt, temperature, humidity, ...}
  Response: {co_next_hour, confidence, aqi_category, features}

GET /api/metrics
  Purpose: Get model performance metrics
  Response: {r2_score, mae, rmse, accuracy, ...}

GET /api/historical?hours=24
  Purpose: Get historical predictions
  Response: {timestamps, actual_co, predicted_co}

GET /api/recommendations?aqi_category=Moderate
  Purpose: Get health recommendations
  Response: {level, description, recommendations, at_risk_groups}

GET /docs
  Purpose: Interactive API documentation (Swagger)
  Response: Beautiful Swagger UI

ML MODEL DETAILS
================================================================================

XGBoost Regressor (CO Prediction)
  Input Features: 22 (current values + lags + rolling means)
  Output: CO concentration for next hour (0-15 mg/m³)
  Performance:
    - R² Score: 0.89
    - MAE: 0.42 mg/m³
    - RMSE: 0.68 mg/m³

  Top Features by Importance:
    1. CO(GT)_lag1           23%
    2. hour                  14%
    3. CO(GT)_rollmean3       9%
    4. NOx(GT)_lag1          6.5%
    5. NOx(GT)_lag3          3.5%
    ... (10 more features)

Random Forest Classifier (AQI Categories)
  Input Features: 6 sensors (NOx, NO2, Temp, Humidity, PT08, Benzene)
  Output: AQI Category
  Performance: 93% Accuracy

  Categories:
    Good:       CO ≤ 2.0 mg/m³    (Green)
    Moderate:   2.0 < CO ≤ 5.0    (Yellow)
    Unhealthy:  5.0 < CO ≤ 10.0   (Orange)
    Hazardous:  CO > 10.0         (Red)

================================================================================
TROUBLESHOOTING
================================================================================

Frontend won't start:
  $ pnpm install        # Install dependencies
  $ pnpm dev            # Try again

Backend won't start:
  $ cd backend
  $ pip install -r requirements.txt
  $ uvicorn main:app --reload

Port already in use:
  pnpm dev -- -p 3001                    (Frontend)
  uvicorn main:app --reload --port 8001  (Backend)

Can't connect frontend to backend:
  - Check backend is running: curl http://localhost:8000/health
  - Check .env.local has correct NEXT_PUBLIC_API_URL
  - Check CORS is enabled in main.py

Models not loading:
  - Models are optional! Dashboard works without them
  - Run training script: python train_models.py --data <csv>
  - Models save to backend/models_pickle/
